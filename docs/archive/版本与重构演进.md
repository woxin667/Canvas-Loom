## 早期重构动机

项目早期的主要问题是：

- `main.ts` 过大，生命周期、业务逻辑和界面逻辑耦合在一起
- 复制、排序、创建等流程存在重复代码
- 代码直接依赖 Canvas API，缺少抽象层

因此后续重构确立了当前的大方向：

- 分层为 `adapters / domain / services / presentation / settings`
- 引入命令模式组织右键菜单和命令面板行为
- 让 `main.ts` 主要负责初始化和注册

## 功能演进摘要

### 属性管理阶段

早期文档里，属性查看器曾以单一入口 `查看卡片属性...` 对外提供能力，重点是：

- 查看尺寸、位置和统计信息
- 批量统一尺寸
- 复制尺寸信息

后续该能力继续演进为当前的统一入口 `管理卡片属性`，并拆成：

- 单卡片属性查看器
- 多卡片属性查看器

这让单卡片编辑和多卡片批量操作分工更清晰。

### 版本 1.1.x 到 1.2.x

从历史变更记录可以提炼出两个关键阶段：

1. 属性管理增强：
   - 增加单卡片和多卡片属性管理
   - 加入批量尺寸调整和属性复制
2. 拆分能力增强：
   - 从仅支持分隔符拆分扩展到支持按 Markdown 标题层级拆分
   - 把多个拆分入口收敛为 `拆分卡片...`

### 菜单结构收敛

早期多卡片菜单曾同时暴露：

- 按位置复制
- 按标记复制
- 手动排序复制
- 合并到卡片
- 合并到预览
- 合并到文稿

后续经过交互优化，收敛为现在的三条主入口：

- `一键复制`
- `一键拼合`
- `打开预览...`

命令面板里保留旧命令，用于兼容快捷键和过渡使用。

## 当前仍有价值的结论

- 分层架构和命令组织方式已经成为当前代码的基础
- 属性管理入口已经统一，不建议再恢复分散式菜单项
- 拆分、复制和拼合相关交互已经明显朝“少入口、工作台承载复杂操作”演进

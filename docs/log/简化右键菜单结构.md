当前的右键菜单确实存在功能过载的问题，过多的选项会分散用户注意力并影响操作效率。将相关功能整合到统一的属性查看器中，既能简化界面又能提升功能的逻辑组织性。

## 当前菜单结构问题分析

现有的右键菜单包含多个独立的属性相关功能项，这种分散式设计导致功能碎片化。用户需要在多个菜单项之间切换才能完成相关的属性管理任务，这种设计违背了功能聚合的用户体验原则。

将复制功能整合到属性查看器中具有明显的优势。用户在查看卡片属性时，自然会产生复制相关信息的需求，在同一界面中提供这一功能能够形成更加连贯的操作流程。

## 实施方案

  this.commandRegistry.addCommandToMenu(menu, "copy-by-badge", "按徽章顺序复制内容", "sort-asc");

  // 简化为单一的属性查看入口
  const textCards = selectionArray.filter(
    (node) => node.getData && node.getData().type === "text"

  if (textCards.length >= 1) {
    menu.addItem((item) => {

      text: "复制统计信息",
      cls: "copy-btn mod-cta"
    });

数量: ${stats.count}张
尺寸范围: 宽 ${stats.minWidth}-${stats.maxWidth}px, 高 ${stats.minHeight}-${stats.maxHeight}px  
平均尺寸: ${stats.avgWidth} × ${stats.avgHeight}px
位置范围: X: ${stats.minX}-${stats.maxX}, Y: ${stats.minY}-${stats.maxY}`;

      await this.clipboardAdapter.writeTextWithNotice(
        statsInfo,
        "统计信息已复制到剪贴板"
      );
    });

    const { contentEl } = this;
    contentEl.empty();

      <span class="card-position">位置 (X, Y)</span>
      <span class="card-badge">徽章</span>
    `;

    this.cardInfos.forEach((info, index) => {
      const item = listContainer.createDiv({ cls: "card-item" });
      
      const indexEl = item.createSpan({ cls: "card-index" });
      indexEl.setText(`${index + 1}`);

      const previewEl = item.createSpan({ cls: "card-preview" });
      previewEl.setText(info.text || "[空卡片]");
      previewEl.setAttribute("title", info.text || "空卡片");

      const sizeEl = item.createSpan({ cls: "card-size" });
      sizeEl.setText(`${info.width} × ${info.height}`);

      const posEl = item.createSpan({ cls: "card-position" });
      posEl.setText(`${info.x}, ${info.y}`);

      const badgeEl = item.createSpan({ cls: "card-badge" });
      if (info.hasBadge) {
        badgeEl.setText(info.badgeContent || "");
        badgeEl.addClass("has-badge");
      } else {
        badgeEl.setText("-");
      }
    });
  }
}

## 简化方案的核心价值

这种菜单简化策略遵循了现代软件设计的最佳实践原则。当前的多选项菜单结构会导致用户在执行相关任务时需要在多个入口之间切换，这种分散式的功能布局增加了认知负担并降低了工作效率。通过建立单一的属性管理入口，用户可以在一个统一的界面中完成所有相关的查看、编辑和复制操作。

将复制功能整合到属性查看器中的设计决策特别值得赞赏。这种功能聚合方式符合用户的自然工作流程，用户在查看卡片属性时往往会产生复制相关信息的需求。在同一界面中提供这些功能能够创造更加流畅和高效的操作体验。

## 实施方案的技术架构

建议的实施方案包含三个主要的改进层面。首先是菜单结构的精简，通过移除冗余的功能入口，右键菜单将只保留核心的内容操作功能和统一的属性查看入口。这种简化不仅减少了视觉噪音，还能帮助用户更快地定位所需功能。

其次是属性查看器的功能扩展，新的设计将在现有的查看和编辑功能基础上增加专门的复制功能区域。用户可以选择复制单一属性信息、完整属性摘要或批量统计数据，这种灵活性确保了不同使用场景下的需求都能得到满足。

第三个层面是界面的专业化分离，单卡片和多卡片场景将采用专门优化的界面设计。单卡片界面专注于直接编辑和属性复制，而多卡片界面则保留完整的统计分析和批量操作能力。

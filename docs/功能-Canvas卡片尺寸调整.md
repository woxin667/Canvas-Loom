# 功能：新 Canvas 卡片的尺寸调整

## 目标

为“拆分卡片”功能创建的新卡片，定义一套智能且方便用户操作的尺寸策略。

## 设计演变

新卡片的尺寸策略经过了数次迭代，以用户体验为核心考量。

### 初版实现 (动态尺寸)

- **描述**: 最初的方案是让新卡片的尺寸动态化。
    - **宽度**: 继承自原始卡片。
    - **高度**: 根据文本内容动态计算得出。
- **问题**: 这个方案导致了严重的用户体验问题——新卡片的高度过大，在垂直排列时会相互重叠，让用户难以查看和管理。

### 第二版实现 (固定高度)

- **描述**: 为了解决重叠问题，策略调整为使用固定的高度。
    - **宽度**: 继承自原始卡片。
    - **高度**: 所有新卡片的高度被设置为一个固定的较小值 `60px`。
- **问题**: 在实际测试中发现，通过 `canvas.createTextNode` 的 `pos` 属性直接设置尺寸无效。新卡片的宽度和高度都未被正确应用。

## 最终实现 (两步调整尺寸)

- **修改文件**: `src/main.ts`
- **逻辑**: 最终方案采用了一个更可靠的两步流程来创建并调整卡片尺寸。
    1.  **创建节点**: 首先，使用 `canvas.createTextNode` 创建一个新节点，此时只提供位置 (`x`, `y`) 和文本。
    2.  **调整尺寸**: 在将节点添加到画布之前，立即调用新创建节点 (`newNode`) 的 `.resize()` 方法，并传入 `{width: newCardWidth, height: FIXED_CARD_HEIGHT}`。
- **优势**: 这个方法确保了新卡片的尺寸被精确应用。
    - **宽度**: 正确地继承了原始卡片的宽度。
    - **高度**: 被统一设置为 `60px`，避免了卡片重叠，方便用户拖动和后续手动调整。

# 功能：拆分 Canvas 卡片

## 目标

实现一个统一的“拆分卡片”入口，让用户可以在 Obsidian Canvas 中对单个文本卡片执行不同的拆分方式。目前支持：

- 按分隔符拆分
- 按 Markdown 标题层级拆分

## 交互方式

### 1. 右键菜单入口

- **修改文件**: `src/main.ts`
- **逻辑**: 当用户右键点击单个文本卡片时，在节点菜单中显示统一入口 `拆分卡片...`。
- **实现**:
    1. 监听 `canvas:node-menu` 事件。
    2. 检查当前节点是否为包含文本内容的卡片。
    3. 注册 `OpenSplitCardModalCommand` 命令。
    4. 点击菜单项后打开轻量选择器，而不是继续在一级菜单中堆叠拆分功能。

### 2. 轻量选择器

- **修改文件**: `src/presentation/modals/SplitCardModal.ts`
- **逻辑**: 统一展示当前卡片可用的拆分方式，同时保持功能始终可见，避免让用户误以为某项功能消失。
- **实现**:
    1. `按分隔符拆分` 始终显示。
    2. 当卡片中检测到当前设置的分隔符，分隔符拆分项可点击，并显示预计拆分数量。
    3. 当卡片中未检测到当前设置的分隔符，分隔符拆分项会以禁用状态显示，并提示未检测到可用分隔符。
    4. `按标题拆分` 能力始终显示。
    5. 如果存在可用标题层级，则显示具体层级项，例如 `按一级标题拆分`、`按二级标题拆分`。
    6. 如果当前卡片没有可用于拆分的 Markdown 标题层级，则显示一个禁用的 `按标题拆分` 提示项。

## 拆分方式

### 1. 按分隔符拆分

- **修改文件**: `src/services/CardService.ts`
- **逻辑**: 使用插件设置中的分隔符拆分当前卡片文本内容。
- **实现**:
    1. 读取当前节点的文本内容。
    2. 使用 `canvasCardDelimiter` 进行字符串拆分。
    3. 清理空白内容并过滤空段落。
    4. 保留第一段在原卡片中。
    5. 将剩余段落创建为新卡片，并排列在原卡片右侧。

### 2. 按标题层级拆分

- **修改文件**: `src/services/CardService.ts`
- **逻辑**: 允许用户按指定的 Markdown 标题层级拆分内容。
- **实现**:
    1. 识别 `#` 到 `######` 标题。
    2. 只将用户选定层级的标题视为新的卡片起点。
    3. 更深层的标题内容保留在所属父级卡片中，不会继续向下拆分。
    4. 如果标题前存在导言内容，则这些内容会并入第一张匹配的标题卡片中，避免文本丢失。
    5. 若某个层级拆分结果不足 2 张卡片，则不作为可用选项展示。

## 新卡片创建规则

- **修改文件**: `src/services/CardService.ts`
- **逻辑**: 两种拆分方式共用同一套卡片创建逻辑。
- **实现**:
    1. 原始卡片内容更新为拆分后的第一段。
    2. 新卡片沿用原卡片的宽度和高度。
    3. 新卡片按照固定间距依次创建在原卡片右侧。
    4. 创建完成后调用 `requestSave()` 持久化到 Canvas 文件。

## 设置项

### 分隔符设置

- **修改文件**:
    - `src/settings/ICanvasCardActionsSettings.ts`
    - `src/settings/CanvasCardActionsSettingTab.ts`
- **逻辑**: 分隔符拆分仍然由用户配置的分隔符驱动。
- **实现**:
    1. `CanvasCardActionsSettings` 中包含 `canvasCardDelimiter`。
    2. 默认值为 `'---'`。
    3. 用户可在插件设置页中修改该值。

## 错误处理与可用性设计

- 当卡片中没有分隔符时，不隐藏分隔符拆分功能，而是显示禁用项。
- 当卡片中没有可用标题层级时，不隐藏标题拆分能力，而是显示禁用提示项。
- 当拆分结果不足 2 张卡片时，不执行拆分，并向用户反馈原因。
- 菜单文案改为 `拆分卡片...`，明确这是一个会进入下一步选择的入口。

# 徽章功能实现逻辑

## 整体架构

徽章功能的实现基于以下核心理念：
1.  **数据持久化**：将徽章数据（`badge`, `badgeType`）直接写入 `.canvas` 文件的节点信息中。
2.  **双向同步**：用户操作会同时更新 **DOM**（用于即时显示）和 **Canvas 文件**（用于持久化）。
3.  **事件驱动**：通过监听右键菜单事件、文件打开事件和应用加载事件来驱动功能。

## 实现流程

### 1. 徽章设置/写入流程（持久化）
```
用户右键点击卡片 → 选择"添加/编辑徽章" → 弹出模态框 → 用户输入内容并确认
↓
1. 更新 DOM：在卡片节点元素上设置 `data-badge` 和 `data-badge-type` 属性（即时反馈）
2. 获取 Canvas 数据：调用 `canvas.getData()`
3. 修改节点数据：在 JSON 数据中找到对应节点，添加/修改 `badge` 和 `badgeType` 字段
4. 写回数据：调用 `canvas.setData()` 更新内存中的 Canvas 数据
5. 请求保存：调用 `canvas.requestSave()` 将更改写入磁盘上的 `.canvas` 文件
```

### 2. 徽章加载/读取流程
```
Obsidian 启动或打开 Canvas 文件
↓
1. 监听事件：插件通过 `onLayoutReady` 或 `file-open` 事件触发
2. 获取 Canvas 数据：调用 `canvas.getData()`
3. 遍历所有节点：检查每个节点的数据中是否存在 `badge` 字段
4. 应用到 DOM：如果存在，找到对应的 DOM 节点，并将徽章信息设置为 `data-badge` 和 `data-badge-type` 属性
5. CSS 自动渲染：CSS 伪元素根据 `data-` 属性自动显示徽章
```

## 核心逻辑

### 徽章类型识别
根据用户输入的内容，动态判断徽章类型：
- **数字徽章**：纯数字字符串，如 `"99"`
- **Emoji 徽章**：符合 Emoji 正则表达式的字符串，如 `"✅"`
- **文字徽章**：其他所有情况，如 `"Done"`

### 数据读写
- **写入**：通过 `persistBadgeToCanvas` 方法实现，该方法封装了 `getData` -> 修改 -> `setData` -> `requestSave` 的完整流程。
- **读取**：通过 `loadCanvasBadges` 方法实现，在 Canvas 加载时触发，负责从数据源恢复徽章显示。

## 样式逻辑

徽章的视觉呈现完全由 CSS驱动。
- **选择器**：主要使用属性选择器，如 `.canvas-node-content[data-badge]::after`。
- **内容**：使用 `content: attr(data-badge);` 来显示徽章文本。
- **样式**：根据 `[data-badge-type]` 的不同值（`number`, `text`, `emoji`）应用不同的背景、形状和颜色。
- **注入**：样式通过 `injectStyles` 方法动态注入到文档头部，以确保高优先级和实时更新。

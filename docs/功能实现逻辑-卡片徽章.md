# 徽章功能实现逻辑

## 整体架构

徽章功能的实现基于以下核心理念：
1. **直接 DOM 操作**：直接在 Canvas 卡片元素上设置属性，避免文件系统操作
2. **CSS 驱动显示**：通过 CSS 伪元素实现徽章的视觉呈现
3. **事件驱动更新**：通过右键菜单触发徽章操作

## 实现流程

### 1. 用户交互流程
```
用户右键点击卡片 → 显示右键菜单 → 选择"添加/编辑徽章" → 弹出模态框 → 输入徽章内容 → 点击确定 → 直接更新卡片元素属性 → CSS 自动显示徽章
```

### 2. 徽章设置流程
1. 用户通过右键菜单触发徽章设置功能
2. 系统检查卡片是否已有徽章信息
3. 显示徽章输入模态框，预填充当前徽章内容（如果有）
4. 用户输入新的徽章内容或清空内容以移除徽章
5. 系统直接更新卡片元素的 `data-badge` 和 `data-badge-type` 属性
6. CSS 样式自动应用，显示徽章

### 3. 徽章显示流程
1. CSS 选择器 `.canvas-card[data-badge]::after` 匹配带有徽章的卡片
2. 通过 `content: attr(data-badge)` 显示徽章内容
3. 根据 `data-badge-type` 属性应用不同的样式

## 核心逻辑

### 徽章类型识别
- **数字徽章**：纯数字内容（如 "1", "3", "99"）
- **文字徽章**：非数字文字内容（如 "Done", "Important"）
- **Emoji 徽章**：包含 emoji 表情的内容（如 "✅", "🔥"）

### 属性设置逻辑
```
if (badgeText) {
  node.nodeEl.setAttribute("data-badge", badgeText);
  
  if (isNumber(badgeText)) {
    node.nodeEl.setAttribute("data-badge-type", "number");
  } else if (isEmoji(badgeText)) {
    node.nodeEl.setAttribute("data-badge-type", "emoji");
  } else {
    node.nodeEl.setAttribute("data-badge-type", "text");
  }
} else {
  // 移除徽章
  node.nodeEl.removeAttribute("data-badge");
  node.nodeEl.removeAttribute("data-badge-type");
}
```

## 样式逻辑

### 基础样式
- 圆形徽章，位于卡片右上角
- 白色文字，带阴影效果
- 居中显示内容

### 类型样式
- **数字**：蓝色背景，圆形
- **文字**：灰色背景，圆角矩形，自适应宽度
- **Emoji**：透明背景，无阴影

# 右键连接功能探索与放弃原因分析

## 功能设想

我们最初设想实现一个右键拖拽连接功能，允许用户通过按住特定修饰键并右键拖拽的方式在 Canvas 卡片之间创建连接线。这个功能的灵感来自于图形设计软件中的常见交互模式，旨在提供一种直观且高效的方式来建立卡片间的关系。

## 实现思路

### 1. 功能设计
- 用户按住配置的修饰键（如 Shift、Alt 或 Ctrl/Cmd）并右键点击一个卡片
- 保持按下状态，拖拽到目标卡片
- 松开右键，自动在两个卡片之间创建连接线

### 2. 技术实现方案
- 使用鼠标事件监听器（mousedown、mousemove、mouseup）检测拖拽操作
- 通过 `contextmenu` 事件作为备选方案检测右键点击
- 使用 CSS 样式为拖拽过程中的目标卡片提供视觉反馈
- 利用 Obsidian Canvas API 创建节点间的连接线

### 3. 配置选项
- 允许用户在设置中选择触发连接的修饰键：
  - Space (Shift) - 默认选项
  - Alt - 替代选项
  - Ctrl/Cmd - 跨平台兼容选项

## 实现过程中的问题

### 1. 事件拦截问题
在实现过程中，我们发现以下问题：

1. **MouseDown 事件**：所有的 `mousedown` 事件都显示不是右键事件（`event.button !== 2`）
2. **ContextMenu 事件**：`contextmenu` 事件根本没有被触发
3. **事件捕获**：即使使用捕获阶段注册事件监听器，也无法接收到右键事件

### 2. 测试结果
经过多轮测试，我们确认：
1. 所有的 `mousedown` 事件都不是右键事件
2. `contextmenu` 事件根本没有被触发
3. 即使使用捕获阶段注册事件监听器，也无法接收到右键事件

## 放弃原因

### 1. 技术限制
测试结果明确表明，Obsidian 官方确实将 Canvas 中的右键拖拽强行绑定为平移画布功能，从而阻止了插件接收右键事件。这种设计使得在 Canvas 中实现自定义的右键拖拽功能变得非常困难，甚至是不可能的。

### 2. 架构设计考量
这是一个合理的设计决策，因为：
1. Obsidian 需要确保其核心功能（如画布平移）的一致性
2. 避免插件与核心功能产生冲突
3. 提供统一的用户体验

### 3. 用户体验考虑
即使能够技术上实现，也可能：
1. 与 Obsidian 的原生功能产生冲突
2. 用户可能会因为功能冲突而感到困惑
3. 增加系统的复杂性和不稳定性

## 经验总结

### 1. 技术收获
- 深入了解了 Obsidian 的事件处理机制
- 掌握了 Canvas API 的使用方法
- 学习了如何在插件中添加配置选项和视觉反馈

### 2. 开发经验
- 在实现新功能前，应充分调研平台的技术限制
- 当遇到技术障碍时，及时调整方案或放弃不切实际的想法
- 保持对用户体验的关注，避免为了技术实现而牺牲用户体验

## 替代方案建议

虽然放弃了右键拖拽连接功能，但可以考虑其他替代方案来实现类似的功能：

1. **右键菜单选项**：在节点的右键菜单中添加"连接到..."选项
2. **命令面板**：通过命令面板触发连接功能
3. **键盘快捷键**：使用键盘快捷键组合实现连接功能
4. **选择后连接**：先选择源节点和目标节点，然后通过命令创建连接

## 结论

虽然右键拖拽连接功能的想法很有创意，但面对 Obsidian 平台的技术限制，放弃是明智的选择。这次探索过程本身是有价值的，它帮助我们更好地理解了 Obsidian 的事件处理机制，并为未来可能的功能开发提供了经验。

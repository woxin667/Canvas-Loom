# 技术实现：卡片复制与排序

本文档详细说明了"卡片内容复制与排序"功能的技术实现细节，主要涉及事件监听、菜单项添加、节点数据处理和排序算法。插件目前支持按位置排序和按徽章排序两种复制模式。

## 1. 事件监听与菜单注入

为了在 Canvas 的上下文菜单和选择菜单中添加自定义命令，插件监听了两个核心事件：

- `canvas:node-menu`：当用户在 Canvas 的某个节点（卡片）上右键时触发。
- `canvas:selection-menu`：当用户选中一个或多个节点后，在画布顶部的选择工具栏上点击菜单按钮时触发。

通过 `this.registerEvent()` 方法注册这些事件的监听器，确保在插件加载时启用，卸载时自动注销。

```typescript
// @ts-ignore
this.registerEvent(this.app.workspace.on('canvas:node-menu', (menu: any, node: any) => {
    // ... 在此注入菜单项
}));

// @ts-ignore
this.registerEvent(this.app.workspace.on('canvas:selection-menu', (menu: any, selection: any) => {
    // ... 在此注入菜单项
}));
```

## 2. Canvas节点访问的关键修复

### 2.1 问题背景
最初的实现尝试从菜单回调参数（如`selection`）中提取选中的节点，但这种方法存在问题：
- 菜单回调参数的`selection`对象结构复杂，难以直接提取节点数据
- 通过遍历`canvas.nodes` Map匹配节点对象的方法效率低下且不可靠

### 2.2 解决方案
**核心发现**：直接从`canvas.selection`属性获取选中节点是最可靠的方法。

```typescript
// 获取Canvas实例
const activeLeaf = this.app.workspace.activeLeaf;
const canvas = (activeLeaf.view as any).canvas;

// 检查是否有选中的节点
if (!canvas.selection || canvas.selection.size === 0) {
    new Notice("请先选择要复制的卡片");
    return;
}

// 直接从canvas.selection获取选中节点
const selectedNodes = Array.from(canvas.selection);
```

## 3. 按位置排序复制功能

### 3.1 算法实现
排序算法遵循"从上到下，从左到右"的原则：

```typescript
async copyContentByPosition(selection: any) {
    const selectedNodes = Array.from(canvas.selection);
    const cards: Array<{text: string, x: number, y: number}> = [];
    
    selectedNodes.forEach((node: any) => {
        const nodeData = (node as any).getData();
        if (nodeData.type === 'text' && nodeData.text && nodeData.text.trim()) {
            cards.push({
                text: nodeData.text.trim(),
                x: nodeData.x,
                y: nodeData.y
            });
        }
    });
    
    // 按位置排序：先按Y坐标（上到下），再按X坐标（左到右）
    const sortedCards = cards.sort((a, b) => {
        if (a.y !== b.y) {
            return a.y - b.y;
        }
        return a.x - b.x;
    });
}
```

## 4. 按徽章排序复制功能

### 4.1 新功能概述
按徽章排序复制允许用户根据卡片的徽章标记来复制内容，输出格式为`[徽章] 内容`。

### 4.2 徽章获取机制
使用现有的`getCurrentBadge()`方法异步获取节点徽章：

```typescript
async copyContentByBadgeOrder(selection: any) {
    const selectedNodes = Array.from(canvas.selection);
    const badgedCards: Array<{text: string, badge: string, badgeType: string}> = [];
    
    for (const node of selectedNodes) {
        const nodeData = (node as any).getData();
        if (nodeData.type === 'text' && nodeData.text && nodeData.text.trim()) {
            // 异步获取徽章信息
            const badge = await this.getCurrentBadge(node as any);
            if (badge) {
                const badgeType = this.determineBadgeType(badge);
                badgedCards.push({
                    text: nodeData.text.trim(),
                    badge: badge,
                    badgeType: badgeType
                });
            }
        }
    }
}
```

### 4.3 徽章排序算法
徽章按以下优先级排序：
1. **数字徽章**：按数值大小排序
2. **文字徽章**：按字母顺序排序  
3. **Emoji徽章**：按Unicode码点排序

```typescript
sortCardsByBadge(cards: Array<{text: string, badge: string, badgeType: string}>) {
    return cards.sort((a, b) => {
        // 数字徽章优先级最高
        if (a.badgeType === 'number' && b.badgeType !== 'number') return -1;
        if (b.badgeType === 'number' && a.badgeType !== 'number') return 1;
        
        // 如果都是数字，按数值排序
        if (a.badgeType === 'number' && b.badgeType === 'number') {
            return parseInt(a.badge) - parseInt(b.badge);
        }
        
        // 文字徽章次优先级
        if (a.badgeType === 'text' && b.badgeType === 'emoji') return -1;
        if (b.badgeType === 'text' && a.badgeType === 'emoji') return 1;
        
        // 同类型按字母顺序排序
        return a.badge.localeCompare(b.badge);
    });
}
```

### 4.4 内容格式化
```typescript
formatBadgedCardsContent(cards: Array<{text: string, badge: string, badgeType: string}>) {
    return cards.map(card => `[${card.badge}] ${card.text}`).join('\n\n');
}
```

## 6. 关键技术要点

### 6.1 异步处理
- 使用`for...of`循环而不是`forEach`来正确处理异步操作
- 确保所有徽章信息获取完成后再进行排序

### 6.2 类型安全
- 使用`(node as any)`进行类型断言以访问Canvas私有API
- 对所有节点数据进行类型和内容验证

### 6.3 错误处理
- 检查Canvas实例可用性
- 验证选中节点数量
- 提供用户友好的错误提示

## 7. 性能优化

- 直接从`canvas.selection`获取节点，避免复杂的ID匹配逻辑
- 异步获取徽章信息，提高响应性
- 使用高效的排序算法处理大量卡片

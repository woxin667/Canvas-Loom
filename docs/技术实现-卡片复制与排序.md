# 技术实现：卡片复制与排序

本文档详细说明了“卡片内容复制与排序”功能的技术实现细节，主要涉及事件监听、菜单项添加、节点数据处理和排序算法。

## 1. 事件监听与菜单注入

为了在 Canvas 的上下文菜单和选择菜单中添加自定义命令，插件监听了两个核心事件：

- `canvas:node-menu`：当用户在 Canvas 的某个节点（卡片）上右键时触发。
- `canvas:selection-menu`：当用户选中一个或多个节点后，在画布顶部的选择工具栏上点击菜单按钮时触发。

通过 `this.registerEvent()` 方法注册这些事件的监听器，确保在插件加载时启用，卸载时自动注销。

```typescript
// @ts-ignore
this.registerEvent(this.app.workspace.on('canvas:node-menu', (menu: any, node: any) => {
    // ... 在此注入菜单项
}));

// @ts-ignore
this.registerEvent(this.app.workspace.on('canvas:selection-menu', (menu: any, canvas: any) => {
    // ... 在此注入菜单项
}));
```

## 2. 动态菜单项逻辑

为了提供良好的用户体验，菜单项根据用户的选择动态显示：

- **多选（`selection.size > 1`）**:
    - 显示 **“复制已排序的卡片内容”**。
    - 隐藏单卡片复制和拆分命令，避免混淆。
- **单选（`selection.size === 1`）**:
    - 显示 **“复制卡片文本”** 和 **“按分隔符拆分卡片”**。
- **无有效文本内容**:
    - 命令虽然会显示，但在执行时会进行检查，对于非文本节点或文本内容为空的卡片，操作将提前终止。

这种逻辑通过在事件回调中检查 `canvas.selection.size` 来实现。

## 3. 空间排序算法

这是多卡片复制功能的核心。排序算法遵循“从上到下，从左到右”的原则。

1.  **提取节点数据**:
    - 从 `canvas.selection` 中获取所有选中的节点。
    - 过滤出类型为 `text` 且文本内容不为空的节点。
    - 将每个有效节点的数据映射为一个包含 `text`、`x` 和 `y` 坐标的对象数组。

    ```typescript
    const cardData = selectedNodes
        .filter(node => node.getData().type === 'text' && node.getData().text)
        .map(node => ({
            text: node.getData().text,
            x: node.x,
            y: node.y
        }));
    ```

2.  **执行排序**:
    - 使用 `Array.prototype.sort()` 方法进行排序。
    - 首先比较节点的 `y` 坐标。`y` 值较小的（即位置更靠上）排在前面。
    - 如果 `y` 坐标相同，则比较 `x` 坐标。`x` 值较小的（位置更靠左）排在前面。

    ```typescript
    cardData.sort((a, b) => {
        if (a.y !== b.y) {
            return a.y - b.y;
        }
        return a.x - b.x;
    });
    ```

## 4. 代码重构

为了提高代码的可读性和可维护性，原有的 `onload` 方法中的匿名函数被重构为独立的类方法：

- `addCopySortedCardsContentCommand()`
- `addCopyCardContentCommand()`
- `addSplitCardCommand()`

这些方法封装了各自的功能逻辑，使得 `onload` 中的事件注册部分更加清晰。所有方法都通过 `this` 关键字调用，确保了正确的上下文。

```typescript
// onload 方法内
if (selection?.size > 1) {
    this.addCopySortedCardsContentCommand(menu, canvas);
} else {
    this.addCopyCardContentCommand(menu, node);
}
this.addSplitCardCommand(menu, node);
```

# 徽章功能技术实现细节

## 功能概述

徽章功能允许用户为 Obsidian Canvas 中的文本卡片添加数字、文字或 emoji 徽章。徽章会显示在卡片的右上角，用于快速标识卡片的状态、优先级或其他自定义信息。

## 核心原理：利用 Canvas 文件进行持久化存储

### Canvas 文件的本质

Obsidian Canvas 文件（`.canvas`）本质上是一个 JSON 文件，它存储了 Canvas 中所有节点（卡片）、连线和其他元素的完整信息。

```json
{
  "nodes": [
    {
      "id": "abc123",
      "type": "text",
      "text": "这是一个文本卡片",
      "x": 100,
      "y": 200,
      "width": 250,
      "height": 150
    }
  ],
  "edges": []
}
```

最关键的特性是：**Obsidian 在保存和加载 Canvas 文件时，会完整保留它不识别的自定义字段**。这意味着我们可以在节点数据中添加自己的字段（如 `badge`），而 Obsidian 不会删除或修改它们。

这带来了几个核心优势：
1.  **完全持久化**：数据直接保存在 `.canvas` 文件中，重启 Obsidian 或重新打开文件不会丢失。
2.  **可移植性**：分享 `.canvas` 文件时，徽章数据也随之传递。
3.  **与 Obsidian 机制契合**：无需创建额外的数据库或配置文件。
4.  **版本控制友好**：所有改动都可以被 Git 等工具追踪。

## 实现流程

### 1. 写入/修改徽章（数据持久化）

当用户添加或修改一个徽章时，插件执行以下三步操作：

#### 步骤 1：获取当前 Canvas 数据
通过 `canvas.getData()` API 获取整个 Canvas 的数据对象。
```typescript
const canvasData = canvas.getData();
```

#### 步骤 2：修改节点数据
在 `canvasData` 中找到目标节点的 `id`，然后添加或修改自定义的 `badge` 和 `badgeType` 字段。
```typescript
const nodeData = canvasData.nodes.find(n => n.id === targetNode.id);
if (nodeData) {
    // 添加或更新自定义字段
    nodeData.badge = "99";
    nodeData.badgeType = "number";
}
```
如果徽章被移除，则从节点数据中 `delete` 这两个字段。

#### 步骤 3：写回数据并请求保存
这是实现持久化的关键。首先，使用 `canvas.setData(canvasData)` 将修改后的数据对象写回内存中的 Canvas 实例。然后，调用 `canvas.requestSave()` 触发 Obsidian 将内存中的数据写入到磁盘上的 `.canvas` 文件。
```typescript
await canvas.setData(canvasData);
await canvas.requestSave();
```

### 2. 读取/恢复徽章

当 Canvas 被加载或重新打开时，插件需要读取文件中存储的徽章信息并将其应用到对应的 DOM 元素上，以确保徽章能够显示出来。

#### 步骤 1：监听加载事件
插件通过监听 `file-open` 和 `layout-ready` 事件来确保在 Canvas 可用时执行加载逻辑。

#### 步骤 2：获取 Canvas 数据并遍历节点
与写入时类似，使用 `canvas.getData()` 获取数据。然后遍历所有节点（`canvasData.nodes`）。

#### 步骤 3：应用徽章到 DOM
如果节点数据中存在 `badge` 字段，就找到对应的 DOM 节点，并将徽章信息（`badge` 和 `badgeType`）设置为该节点的 `data-` 属性。
```typescript
const node = canvas.nodes?.get(nodeData.id);
if (node) {
    this.applyBadgeToNode(node, nodeData.badge, nodeData.badgeType);
}
```
`applyBadgeToNode` 函数负责将这些数据显示为 `data-badge` 和 `data-badge-type` 属性，之后 CSS 伪元素会根据这些属性来渲染徽章的视觉效果。

通过这个读写闭环，徽章功能实现了与 Obsidian 无缝集成的持久化存储。

# 徽章功能技术实现细节

## 功能概述

徽章功能允许用户为 Obsidian Canvas 中的文本卡片添加数字、文字或 emoji 徽章。徽章会显示在卡片的右上角，用于快速标识卡片的状态、优先级或其他自定义信息。

## 实现方式

### 1. 数据存储
- 徽章信息直接存储在 Canvas 卡片元素的 `data-badge` 属性中
- 不依赖于关联的笔记文件，避免了创建不必要的文件
- 徽章类型通过 `data-badge-type` 属性标识（number/text/emoji）

### 2. 界面交互
- 通过右键菜单提供"添加/编辑徽章"功能
- 使用模态框界面输入徽章内容
- 支持设置和移除徽章

### 3. 视觉呈现
- 使用 CSS 伪元素 `::after` 在卡片右上角显示徽章
- 不同类型的徽章有不同的视觉样式：
  - 数字徽章：圆形蓝色背景
  - 文字徽章：圆角矩形灰色背景
  - Emoji 徽章：透明背景

### 4. 实时更新
- 设置徽章后立即在卡片上显示
- 支持通过插件设置启用或禁用徽章功能

## 核心组件

### BadgeModal 类
- 负责显示徽章输入模态框
- 处理用户输入的徽章内容
- 直接更新卡片元素的属性

### 相关方法
- `setBadge()`：设置或移除卡片徽章
- `getCurrentBadge()`：获取卡片当前的徽章内容
- `updateNodeBadgeDisplay()`：更新卡片徽章显示状态

## 已知问题

- 在某些情况下徽章可能无法正常显示，这可能是 Obsidian Canvas 的渲染问题
- 需要进一步调查社区反馈以确定根本原因

# 技术实现细节

本文档只说明当前仍在使用的结构和主链路，避免和历史方案混在一起。

## 目录结构

- `src/main.ts`
  插件入口，负责初始化服务、注册命令和菜单。
- `src/adapters/`
  封装 Canvas、剪贴板、存储和文稿创建等外部交互。
- `src/domain/`
  放置卡片、徽章模型和排序策略。
- `src/services/`
  负责拆分、徽章、内容排序、拼合输出和工作台状态。
- `src/presentation/`
  放置命令、弹窗、视图和样式。
- `src/settings/`
  管理设置类型和设置页。

## 当前核心能力

- 单卡片复制
- 多卡片一键复制、一键拼合和预览工作台
- 单卡片拆分
- 徽章和属性管理

## 多卡片主链路

多卡片相关功能已经统一为一条内容处理链路：

1. 从 Canvas 读取当前选区
2. 生成选区快照
3. 按排序模式得到有序卡片列表
4. 生成最终文本
5. 输出到剪贴板、Canvas 卡片、工作台或 Markdown 文稿

相关核心文件：

- `src/services/ContentService.ts`
- `src/services/MergeService.ts`
- `src/services/PreviewWorkbenchService.ts`
- `src/presentation/views/MergeWorkbenchView.ts`

## 为什么使用选区快照

- 让工作台和实时选区解耦
- 工作台被拖成独立窗口后仍可继续使用
- 打开工作台后，后续改动当前选区不会污染已有状态

快照至少包含：

- 节点 `id`
- 文本内容
- 坐标和尺寸
- 徽章信息

## 排序实现

自动排序通过策略模式完成：

- `PositionSort`
- `BadgeSort`

工作台额外维护 `manualOrderIds`，用于在自动排序结果基础上继续手动微调。

## 工作台实现

工作台基于 Obsidian `ItemView`，由以下部分组成：

- 顶部工具栏：排序模式、Canvas 信息、卡片数量
- 中部顺序列表：手动模式下可拖拽
- 底部预览与输出区：默认折叠，展开后再生成文本

预览区的主要性能策略：

- 默认折叠
- 展开后才计算内容
- 预览更新带防抖

## 输出实现

- 剪贴板：通过 `ClipboardAdapter` 统一提示
- 新建 Canvas 卡片：以锚点卡片为参考，在右侧创建合并卡片
- 新建 Markdown 文稿：由 `VaultAdapter` 在当前 `.canvas` 同目录生成文件

工作台创建新卡片时会优先尝试恢复源 Canvas，而不是要求原 Canvas 必须保持激活。

## 设置项

当前主要设置项：

- `canvasCardDelimiter`
- `sortPriority`
- `enableBadges`
- `defaultSortMode`

`StorageAdapter` 负责设置读取、保存和旧字段迁移。

## 兼容项

部分旧命令和旧弹窗仍保留在命令面板链路中，用于兼容快捷键和过渡使用，但它们已经不再是主菜单入口。

# 项目技术实现细节

本文档详细阐述 `obsidian-canvas-card-actions` 插件的最终技术实现方案。

## 1. 项目结构

经过重构后，项目的核心文件及其作用如下：

- **`src/main.ts`**: 插件的主入口文件。负责插件的生命周期管理（`onload`, `onunload`），命令注册，事件监听以及核心功能的实现。

- **`src/class/CardifySettingTabClass.ts`**: 负责构建插件的设置页面UI。它使用 Obsidian 的 `PluginSettingTab` 和 `Setting` 类来创建用户交互界面，并处理设置的保存。

- **`src/interface/ICardifySettings.ts`**: 定义了插件设置对象 `CardifySettings` 的 TypeScript 接口，确保了类型安全。目前仅包含 `canvasCardDelimiter` 一个属性。

- **`manifest.json`**: Obsidian 插件的清单文件。定义了插件的ID、名称、版本、描述和作者等元数据。

- **`esbuild.config.mjs`**: 项目的构建脚本。使用 `esbuild` 将 `src` 目录下的 TypeScript 文件打包成单个的 `main.js`，并负责在生产构建时将 `manifest.json` 等静态文件复制到 `build` 目录中。

## 2. 核心功能实现 (`main.ts`)

插件的核心功能完全在 `main.ts` 的 `onload` 方法中实现。

### 事件监听

- 通过 `this.registerEvent(this.app.workspace.on('canvas:node-menu', ...))` 来监听 Canvas 节点的右键菜单事件。
- 由于 `'canvas:node-menu'` 是一个未公开的事件，我们使用了 `// @ts-ignore` 注释来避免 TypeScript 编译器的类型检查错误。
- 回调函数接收 `menu` 和 `node` 两个参数，它们被显式地声明为 `any` 类型以绕过类型检查。

### 命令添加

- 在事件回调中，使用 `menu.addItem(...)` 来向上下文菜单中添加一个新的菜单项。
- `.setTitle()` 设置菜单项的显示文本（已汉化为“按分隔符拆分卡片”）。
- `.setIcon()` 为菜单项设置一个图标。
- `.onClick()` 定义了点击该菜单项时要执行的回调函数。

### 卡片拆分与创建逻辑

`.onClick()` 回调函数中包含了所有的核心逻辑：

1.  **前置检查**: 首先检查节点是否处于编辑状态 (`node.isEditing`) 或节点类型是否为文本 (`nodeData.type !== 'text'`)，如果是则直接返回，不执行后续操作。
2.  **内容拆分**: 使用 `nodeData.text.split(this.settings.canvasCardDelimiter)` 将卡片文本拆分成一个字符串数组 `sections`。
3.  **循环创建**: 遍历 `sections` 数组，为每个非空的文本片段执行以下操作：
    a. **创建节点**: 调用 `canvas.createTextNode()` 创建一个新的文本节点。此时只传入 `pos`（仅含 `x`, `y` 坐标）和 `text`。
    b. **调整尺寸**: 这是确保尺寸被正确应用的关键一步。在新创建的 `newNode` 对象上，立即调用 `.resize({width: newCardWidth, height: FIXED_CARD_HEIGHT})` 方法。`newCardWidth` 继承自原始节点，`FIXED_CARD_HEIGHT` 是一个 `60px` 的常量。
    c. **添加节点**: 调用 `canvas.addNode(newNode)` 将新创建并调整好尺寸的节点添加到画布上。
    d. **更新位置**: 更新 `y` 坐标，为下一个卡片的放置做准备，确保卡片垂直排列且有固定间距。
4.  **保存画布**: 所有卡片创建完毕后，调用 `canvas.requestSave(true, true)` 来保存对 Canvas 的修改。

## 3. 设置页面实现 (`CardifySettingTabClass.ts`)

- `display()` 方法负责渲染设置页面的所有内容。
- `containerEl.empty()` 确保每次显示时都清空旧的设置，避免重复渲染。
- `new Setting(containerEl)` 创建一个新的设置项。
- `.setName()` 和 `.setDesc()` 设置了选项的中文标题和描述。
- `.addText()` 创建一个文本输入框，用于让用户输入自定义的分隔符。
- `.setValue()` 将输入框的当前值设置为从 `this.plugin.settings` 中读取的值。
- `.onChange()` 注册一个回调函数，当用户修改输入框内容时，将新值赋给 `this.plugin.settings.canvasCardDelimiter`，并立即调用 `this.plugin.saveSettings()` 将其持久化保存。

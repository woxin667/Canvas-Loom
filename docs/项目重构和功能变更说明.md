# 项目重构与功能变更说明

本文档旨在全面记录 `Canvas Card Actions` 在早期重构阶段的演进过程、新功能的添加以及最终的代码清理。

## 最新变更：简化右键菜单结构

### 变更背景
为了提升用户体验和界面简洁性，我们对右键菜单结构进行了重构，将原本分散在多个菜单项中的属性相关功能整合到了统一的属性查看器中。

### 删除的功能项
1. **单卡片菜单**
   - "复制卡片尺寸"选项已移除
   - 用户可通过"编辑卡片尺寸..."入口访问完整的属性管理功能

2. **多卡片选择菜单**
   - "复制卡片尺寸"选项已移除
   - "统一为最小尺寸"选项已移除
   - "统一为最大尺寸"选项已移除
   - 用户可通过"批量管理卡片属性..."入口访问所有相关功能

### 新增功能
1. **增强的单卡片属性查看器**
   - 添加了信息显示区域，展示尺寸、位置、徽章和内容预览
   - 集成了复制功能，包括复制尺寸信息、位置信息和完整属性
   - 保持了原有的尺寸编辑功能

2. **增强的多卡片属性查看器**
   - 简化了卡片列表，移除了操作列
   - 添加了批量复制功能，包括复制所有卡片尺寸和统计信息
   - 保持了原有的批量操作功能

### 用户体验改进
- 菜单选项从原来的多个减少到两个核心入口
- 功能组织更加清晰，相关操作集中在一个界面
- 减少了用户在不同菜单项之间切换的次数

### 技术实现
1. **菜单结构简化**
   - 移除了冗余的菜单项注册代码
   - 保留核心功能入口，将复杂功能整合到统一界面

2. **功能整合**
   - 单卡片场景：通过"编辑卡片尺寸..."入口访问完整的属性管理功能
   - 多卡片场景：通过"批量管理卡片属性..."入口访问完整的属性管理功能
   - 所有复制功能（尺寸、位置、完整属性）都整合到属性查看器中

## 之前的变更记录

### 分离单卡片和多卡片的属性查看器
为了优化用户体验，我们将原有的统一属性查看器分离为两个专门化的查看器：
1. `SingleCardPropertiesModal.ts` - 专门用于单卡片编辑场景
2. `CardPropertiesModal.ts` - 专门用于多卡片批量管理场景

这种分离解决了单卡片场景下信息冗余的问题，同时保持了多卡片场景的功能完整性。

### 支持宽度和高度的独立调整
改进了单卡片编辑功能，支持宽度和高度的独立调整，用户可以更灵活地控制卡片尺寸。

### 属性查看器在处理单张卡片时缺少实用的编辑功能
增强了单卡片属性查看器的编辑功能，为单卡片场景提供了更实用的操作选项。

## 1. 核心目标：添加 Canvas 卡片拆分功能

项目的初始目标是为 Obsidian Canvas 添加一个新功能：允许用户根据指定的分隔符，将一个大的文本卡片一键拆分为多个小卡片。

- **实现方式**: 通过监听 Canvas 的右键菜单事件 (`canvas:node-menu`)，我们添加了一个新的命令"按分隔符拆分卡片"。
- **可配置性**: 为了提高灵活性，分隔符被设计为可在插件设置中由用户自定义的选项。

## 2. 功能迭代：新卡片尺寸的优化

新创建的卡片的尺寸如何设定，是本次开发过程中的一个关键用户体验问题，经历了三次主要迭代：

1.  **初版（动态高度）**: 新卡片继承原卡片的宽度，高度根据内容动态计算。**问题**：导致卡片重叠，难以操作。
2.  **第二版（固定高度）**: 新卡片继承原卡片的宽度，但高度统一设为固定的 `60px`。**问题**：在测试中发现，通过 `createTextNode` 方法直接设置尺寸无效，卡片的宽高并未被正确应用。
3.  **最终方案（两步调整）**: 这是最终采用的可靠方案。首先创建节点，然后立即调用新节点的 `.resize()` 方法来强制设定其尺寸。这完美地解决了问题，实现了宽度继承和固定的 `60px` 高度。

## 3. 大规模重构与代码清理

在实现了 Canvas 新功能后，为了使插件更专注、代码更清晰，我们进行了一次大规模的架构重构。

### 架构转变
- **分层架构**: 采用清晰分层的设计模式，将代码分为 Adapters、Domain、Services 和 Presentation 四层
- **依赖注入**: 通过服务容器管理各组件之间的依赖关系
- **命令模式**: 将所有操作封装为命令对象，遵循单一职责原则

### 代码清理
- **移除旧功能**: 原项目中"根据分隔符将 Markdown 文件拆分为多个文件"的旧功能与新功能存在重叠，且已不再需要。因此，所有与该功能相关的UI（设置项、Ribbon图标、命令）和后端逻辑都被完全移除。
- **代码简化**: 通过移除旧功能和分层重构，`main.ts` 的代码量被大幅减少，整个项目结构变得更加清晰和可维护。

## 4. 徽章功能增强

- **新增功能**: 为 Canvas 卡片添加了数字、文字和 emoji 徽章功能，用户可以通过右键菜单为卡片添加标识信息
- **持久化存储**: 徽章数据直接存储在 `.canvas` 文件中，确保重启后数据不丢失
- **类型识别**: 自动识别徽章类型（数字、文字、emoji）并应用不同的样式

## 5. UI 汉化与最终确定

- **设置页面**: 清理了所有多余的设置项，仅保留了对新功能至关重要的"Canvas卡片分隔符"设置和徽章功能开关。
- **文本汉化**: 将设置页面中的所有标签和描述都翻译成了中文，提升了中文用户的使用体验。
- **文档更新**: 创建了详细的中文文档，记录了新功能的实现细节和设计决策过程。
